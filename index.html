Refining an AI-generated landing page into production quality

AI tools produce landing pages fast — and almost-right. This is the exact review pass I run to close the gap: responsiveness, semantics, accessibility, SEO, and performance. Every fix below is real code from this refactor.

1. Fixed-width layouts that break on mobile

The AI draft hard-coded width: 1150px on every section — navbar, hero, features, footer. It looked perfect at 1440px and forced horizontal scrolling on anything narrower than a laptop.

✗ AI draft

.navbar { width: 1150px; margin: 0 auto; }
.hero   { width: 1150px; display: flex; }
.hero-left { width: 560px; }
.hero-title { font-size: 56px; }
.feature-card { width: 363px; }

✓ Production

.container { max-width: 1150px;
  margin: 0 auto; padding: 0 20px; }
.hero { display: flex;
  flex-wrap: wrap; gap: 48px; }
.hero-left { flex: 1 1 420px; }
.hero-title {
  font-size: clamp(32px, 6vw, 56px); }

Why it matters: max-width + flex-wrap + clamp() gives one layout system that works from 360px to 4K — no per-breakpoint rewrites, no horizontal scroll.

2. Div soup instead of semantic HTML

Everything was a <div> — including the navigation, headings, and buttons. Screen readers see no structure; search engines see no hierarchy.

✗ AI draft

<div class="nav-link">Pricing</div>
<div class="hero-title">
  Manage Projects…</div>
<div class="btn-primary"
  onclick="alert('Sign up
  coming soon!')">
  Start Free Trial</div>

✓ Production

<nav aria-label="Main navigation">
  <a href="#pricing">Pricing</a>
</nav>
<h1>Manage Projects…</h1>
<a href="#signup" class="btn-primary">
  Start Free Trial</a>

Why it matters: real <nav>, <h1>–<h3> hierarchy, and links-as-links restore keyboard navigation, screen-reader landmarks, and crawlable structure. Semantic and contrast fixes took accessibility from 62 to 100.

3. Missing SEO & social metadata

The draft shipped with <title>Studioflow</title> and nothing else. No meta description, no Open Graph, no canonical — and no lang attribute on the html element.

✓ Added

<html lang="en">
<title>Studioflow — Project Management for Creative Teams</title>
<meta name="description" content="Studioflow is the all-in-one project management platform…">
<meta property="og:title" content="Studioflow — Project Management for Creative Teams">
<meta property="og:description" content="Boards, real-time collaboration, and analytics…">
<link rel="canonical" href="https://airefactorcasestudy.vercel.app/refactored.html">

Why it matters: pages built to run ads against live or die on link previews and search snippets. SEO score 82 → 100.

4. Performance: render-blocking everything

A render-blocking @import font with 4 weights, and three 2400×1600 images loaded at full resolution with no dimensions and no lazy loading.

  • ✓Images requested at 800×533 (their rendered size) with explicit width/height to prevent layout shift
  • ✓fetchpriority="high" on the hero image, loading="lazy" on everything below the fold
  • ✓Font @import → <link> with preconnect and display=swap; 4 weights → 3
  • ✓Descriptive alt text on every image (was empty)

Result: Performance 88 → 96, with LCP as the remaining bottleneck to watch on image-heavy pages.

5. Forms that only pretended to work

The AI's email form had no labels, no validation, and no submit handling — a styled dead end.

✗ AI draft

<input type="text" class="email-input"
  placeholder="Enter your email">
<div class="btn-primary" onclick=
  "alert('Thanks for signing up!')">
  Sign Up Free</div>

✓ Production

<form class="signup-form">
  <label class="sr-only" for="email">
    Work email</label>
  <input id="email" name="email"
    type="email" required
    autocomplete="email"
    placeholder="Enter your work email">
  <button type="submit"
    class="btn-primary">
    Sign Up Free</button>
</form>

Why it matters: a real endpoint, native validation, an accessible label, and autocomplete. Conversion elements have to convert.

My AI-collaboration checklist

AI output is a strong first draft, not a deliverable. Before anything ships, I verify:

① Mobile-first at 360 / 768 / 1280px

② Semantic landmarks & heading order

③ Keyboard & screen-reader pass

④ Meta, OG, canonical present

⑤ Images sized, compressed, lazy

⑥ Forms wired, validated, labeled

⑦ Lighthouse ≥ 95 across the board

⑧ Copy placement matches the brief

This is exactly the "human-in-the-loop" role: AI for velocity, human judgment for quality.
